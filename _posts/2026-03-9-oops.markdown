Object-Oriented Programming (OOP) is a paradigm that organizes software around “objects” — entities that combine data (attributes) and behavior (methods). The four pillars of OOP are Encapsulation, Abstraction, Inheritance, and Polymorphism.


Core OOP Concepts


Encapsulation: This principle binds data and methods together within a class and restricts direct access to internal data. It is achieved using access modifiers (like private, protected, public in Java). For example, a BankAccount class might hide the balance field, allowing access only through public methods like deposit() and withdraw(), ensuring data integrity. 

Abstraction: Abstraction focuses on exposing only essential features while hiding complex implementation details. It simplifies interaction by providing a clean interface. For instance, when using a car, you only need to know how to use the steering wheel and pedals—details like engine mechanics are abstracted away. 


Inheritance: Inheritance allows a class to inherit properties and methods from another class, promoting code reuse. The class that inherits is called a subclass (or derived class), and the one being inherited from is the superclass (or base class).  For example, a Dog class can inherit from an Animal class, gaining common behaviors like eat() and sleep(), while adding its own like bark(). 


Polymorphism: Polymorphism enables objects of different types to be treated as instances of the same type through a common interface. It allows one interface to represent multiple underlying forms. This is achieved through method overriding (runtime polymorphism) and method overloading (compile-time polymorphism).  For example, a Shape class might have a draw() method, which is overridden in subclasses like Circle and Rectangle to provide specific drawing logic. 


Advanced Concepts (Beyond the Core Four)


Association: Represents a relationship between two classes where objects of one class are related to objects of another. For example, a Teacher is associated with multiple Students. 
Aggregation: A special form of association where one class contains another class, but the contained object can exist independently. This is a "has-a" relationship. For example, a Car has an Engine, but the Engine can exist without the Car. 


Composition: A stronger form of aggregation where the contained object cannot exist independently of the container. It represents a "part-of" relationship. For example, a House is composed of Rooms; if the House is destroyed, the Rooms cease to exist.
